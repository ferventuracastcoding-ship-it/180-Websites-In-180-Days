🚀 Dijkstra Algorithm Visualizer

Button Explanations

▶ Run Dijkstra

Starts the Dijkstra shortest path algorithm. The algorithm explores nearby nodes by choosing the node with the smallest distance value. Blue nodes represent visited locations. When the destination is found, the shortest path is highlighted in yellow.

🧱 Generate Walls

Creates random obstacles on the grid. Walls represent blocked areas that the algorithm cannot travel through. This simulates real-world problems like roads, terrain, or obstacles.

🔄 Reset

Clears all walls, visited nodes, and paths. It creates a fresh grid so you can test another shortest path search.

How Dijkstra's Algorithm Works

Dijkstra's algorithm is a graph searching algorithm used to find the shortest path between two points. The algorithm starts at the green starting node and assigns it a distance of 0. Every other node begins with infinity distance. The algorithm repeatedly chooses the closest unvisited node, checks its neighbors, and updates their distances. Once the red destination node is reached, the algorithm follows the stored previous nodes to rebuild the shortest route.
